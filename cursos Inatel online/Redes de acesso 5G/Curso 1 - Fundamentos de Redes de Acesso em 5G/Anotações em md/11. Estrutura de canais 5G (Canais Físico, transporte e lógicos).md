### Estrutura de canais 5G NR
- O conteúdo fala sobre a estrutura de canais no 5G NR
- A ideia principal é entender como os dados e as informações de controle são organizados antes de serem transmitidos pela interface aérea
- A estrutura é dividida em três categorias principais
	- Canais lógicos
	- Canais de transporte
	- Canais físicos
- Cada categoria tem uma função diferente e trabalha junto com as outras para garantir uma transmissão eficiente, confiável e flexível
### Introdução
- A estrutura de canais no 5G NR é organizada de forma hierárquica
- Essa organização ajuda a separar o tipo de informação, a forma de transporte e a transmissão física pelo rádio
- Os canais lógicos indicam o tipo de dado que está sendo transportado
- Os canais de transporte definem como esses dados serão levados pela interface de rádio
- Os canais físicos representam a transmissão real no meio físico
- Essa divisão é importante porque o 5G precisa atender diferentes cenários, como baixa latência, alta taxa de dados e conexão massiva de dispositivos
### Ideia geral da hierarquia de canais
- A hierarquia dos canais pode ser entendida em três níveis
	- Primeiro os dados são classificados nos canais lógicos
	- Depois são organizados nos canais de transporte
	- Por fim são transmitidos pelos canais físicos
- Essa estrutura permite que o 5G trate controle e tráfego de usuário de formas diferentes
- Também facilita o mapeamento dos dados para os recursos de rádio disponíveis
### Canais lógicos
- Os canais lógicos definem o tipo de informação que está sendo transportada
- Eles dizem se a informação é de controle ou de tráfego do usuário
- Os canais lógicos ficam entre a camada RLC e a camada MAC
- Eles não definem ainda como o dado será transmitido fisicamente
- A função principal é classificar a natureza da informação
- Conceito que eu preciso lembrar
	- Canal lógico responde à pergunta: que tipo de informação está sendo transportada
### Canais lógicos de controle
- Os canais lógicos de controle transportam informações relacionadas ao gerenciamento e à configuração da rede
- Eles carregam sinalização necessária para manter a comunicação funcionando
- Esse tipo de canal é usado para mensagens de controle, acesso, configuração e gerenciamento da conexão
- Exemplos de canais lógicos de controle no downlink
	- BCCH
	- PCCH
	- CCCH
	- DCCH
	- MCCH
- Exemplos de canais lógicos de controle no uplink
	- CCCH
	- DCCH
### Canais lógicos de tráfego
- Os canais lógicos de tráfego transportam dados do usuário
- Esses dados podem incluir voz, vídeo, internet, arquivos e aplicações
- Eles estão ligados ao plano de usuário
- O objetivo é levar o conteúdo real que o usuário está enviando ou recebendo
- Exemplos de canais lógicos de tráfego
	- DTCH
	- MTCH
- Conceito que eu preciso lembrar
	- Canal de controle leva sinalização
	- Canal de tráfego leva dados do usuário
### Canais de transporte
- Os canais de transporte ficam entre a camada MAC e a camada física
- Eles fazem a ponte entre os canais lógicos e os recursos físicos da rede
- A função deles é indicar como os dados serão transportados pela interface de rádio
- Eles especificam características como multiplexação, codificação, modulação e estrutura dos pacotes
- Conceito que eu preciso lembrar
	- Canal de transporte responde à pergunta: como o dado será transportado
### Função dos canais de transporte
- Os canais de transporte pegam as informações vindas dos canais lógicos e organizam para transmissão
- Eles ajudam a adaptar os dados ao tipo de transmissão necessário
- Também participam do mapeamento para os canais físicos
- No 5G, essa etapa é importante porque diferentes serviços exigem tratamentos diferentes
- Um serviço de baixa latência pode precisar de transporte mais rápido
- Um serviço de alta capacidade pode precisar de maior eficiência espectral
### Canais físicos
- Os canais físicos representam a transmissão real pela interface aérea
- Eles ficam na camada PHY
- São responsáveis por levar os dados no meio físico usando sinais de rádio
- Nessa etapa aparecem processos como modulação, correção de erros, multiplexação e alocação de frequências
- Um canal de transporte pode ser mapeado para um ou mais canais físicos
- Conceito que eu preciso lembrar
	- Canal físico responde à pergunta: por onde o dado é transmitido no rádio
### Relação entre canais lógicos, transporte e físicos
- A relação entre os canais segue uma ordem
	- Canais lógicos classificam os dados
	- Canais de transporte organizam como os dados serão levados
	- Canais físicos fazem a transmissão real no rádio
- Essa separação torna a rede mais organizada e permite maior flexibilidade
- O mapeamento entre esses canais é essencial para que a rede entregue dados com qualidade e eficiência
### Mapeamento de downlink
- Downlink é o sentido da rede para o usuário
- No mapeamento de downlink, os canais lógicos descem para canais de transporte e depois para canais físicos
- Canais lógicos mostrados no downlink
	- BCCH
	- PCCH
	- CCCH
	- DCCH
	- MCCH
	- DTCH
	- MTCH
- Canais de transporte mostrados no downlink
	- BCH
	- PCH
	- DL SCH
	- MCH
- Canais físicos mostrados no downlink
	- PBCH
	- PDSCH
	- PDCCH
### BCCH
- BCCH significa Broadcast Control Channel
- É um canal lógico de controle
- Transporta informações de broadcast da célula
- Essas informações ajudam o UE a conhecer parâmetros importantes da rede
- Normalmente é associado a informações do sistema
### PCCH
- PCCH significa Paging Control Channel
- É um canal lógico de controle
- É usado para paging
- Paging é o procedimento usado para chamar ou localizar um UE quando a rede precisa estabelecer comunicação
- É importante quando o dispositivo não está transmitindo dados ativamente
### CCCH
- CCCH significa Common Control Channel
- É um canal lógico de controle comum
- É usado quando ainda não existe uma conexão dedicada estabelecida
- Pode aparecer tanto no downlink quanto no uplink
- Ajuda em procedimentos iniciais de acesso e sinalização
### DCCH
- DCCH significa Dedicated Control Channel
- É um canal lógico de controle dedicado
- Transporta mensagens de controle específicas de um UE
- É usado quando já existe uma conexão dedicada entre o usuário e a rede
### DTCH Dedicated Traffic Channel
- O DTCH é um canal lógico de tráfego dedicado
- Ele é usado para transportar dados do usuário durante uma conexão dedicada
- Esses dados podem ser internet, voz, vídeo, aplicativos ou qualquer tráfego real do usuário
- Diferente dos canais de controle, o DTCH não carrega sinalização da rede
- Ele aparece tanto no downlink quanto no uplink
- No downlink, pode transportar dados da rede para o UE
- No uplink, pode transportar dados do UE para a rede
- Conceito que eu preciso lembrar
    - DTCH é canal lógico dedicado para tráfego de usuário
    - DCCH é canal lógico dedicado para controle
    - DTCH leva dados
    - DCCH leva sinalização
### MCCH
- MCCH significa Multicast Control Channel
- É um canal lógico de controle usado para informações de multicast
- Ele carrega controle relacionado a serviços enviados para múltiplos usuários
- Aparece no mapeamento de downlink
### DTCH
- DTCH significa Dedicated Traffic Channel
- É um canal lógico de tráfego dedicado
- Transporta dados de usuário de forma dedicada
- Pode aparecer no downlink e no uplink
- É usado para dados como internet, voz, vídeo e aplicações
### MTCH
- MTCH significa Multicast Traffic Channel
- É um canal lógico de tráfego multicast
- Transporta dados destinados a múltiplos usuários
- Aparece no mapeamento de downlink
### BCH
- BCH significa Broadcast Channel
- É um canal de transporte
- Carrega informações de broadcast essenciais da rede
- No downlink, pode ser mapeado para o PBCH
### PCH
- PCH significa Paging Channel
- É um canal de transporte relacionado ao paging
- Ele transporta mensagens usadas para localizar ou chamar o UE
- No downlink, aparece entre os canais de transporte
### DL SCH
- DL SCH significa Downlink Shared Channel
- É um canal de transporte compartilhado de downlink
- Transporta dados e informações para usuários no sentido rede para UE
- É um dos canais mais importantes para tráfego de dados no downlink
- Pode ser mapeado para o PDSCH
### MCH
- MCH significa Multicast Channel
- É um canal de transporte usado para transmissão multicast
- Ele transporta dados destinados a múltiplos usuários
- Aparece no mapeamento de downlink
### PBCH
- PBCH significa Physical Broadcast Channel
- É um canal físico de downlink
- Transmite informações essenciais de broadcast
- Está associado ao acesso inicial do UE à rede
- Ele carrega informações necessárias para que o dispositivo consiga identificar e acessar a célula
### PDSCH
- PDSCH significa Physical Downlink Shared Channel
- É um canal físico de downlink
- Transporta dados de usuário e algumas informações de controle no sentido rede para UE
- É o principal canal físico usado para dados no downlink
- Está relacionado ao DL SCH
### PDCCH
- PDCCH significa Physical Downlink Control Channel
- É um canal físico de controle no downlink
- Transporta informações de controle necessárias para o UE saber onde e como receber ou enviar dados
- Ele indica alocação de recursos, agendamento e informações de controle
- Conceito que eu preciso lembrar
	- PDCCH carrega controle
	- PDSCH carrega dados de downlink
### Mapeamento de uplink
- Uplink é o sentido do usuário para a rede
- No mapeamento de uplink, os canais lógicos são mapeados para canais de transporte e depois para canais físicos
- Canais lógicos mostrados no uplink
	- CCCH
	- DCCH
	- DTCH
- Canais de transporte mostrados no uplink
	- UL SCH
	- RACH
- Canais físicos mostrados no uplink
	- PUSCH
	- PUCCH
	- PRACH
### UL SCH
- UL SCH significa Uplink Shared Channel
- É um canal de transporte compartilhado de uplink
- Transporta dados enviados pelo UE para a rede
- Pode carregar tráfego de usuário e algumas informações associadas
- É mapeado para o PUSCH
### RACH
- RACH significa Random Access Channel
- É um canal de transporte usado no procedimento de acesso aleatório
- Ele permite que o UE inicie comunicação com a rede
- É importante em situações como acesso inicial, recuperação de conexão ou sincronização
- É mapeado para o PRACH
### PUSCH
- PUSCH significa Physical Uplink Shared Channel
- É um canal físico de uplink
- Transporta dados do usuário no sentido UE para rede
- Está relacionado ao UL SCH
- É o principal canal físico de dados no uplink
### PUCCH
- PUCCH significa Physical Uplink Control Channel
- É um canal físico de controle no uplink
- Transporta informações de controle enviadas pelo UE para a rede
- Pode carregar confirmações, informações de qualidade do canal e outros sinais de controle
- Conceito que eu preciso lembrar
	- PUCCH carrega controle no uplink
	- PUSCH carrega dados no uplink
### PRACH
- PRACH significa Physical Random Access Channel
- É o canal físico usado para acesso aleatório
- Ele permite que o UE solicite acesso inicial à rede
- É usado quando o dispositivo precisa iniciar ou recuperar comunicação
- Está ligado ao RACH
### Comparação rápida entre downlink e uplink
- Downlink
	- Sentido da rede para o usuário
	- Usa canais físicos como PBCH, PDSCH e PDCCH
	- PDCCH leva controle
	- PDSCH leva dados
- Uplink
	- Sentido do usuário para a rede
	- Usa canais físicos como PUSCH, PUCCH e PRACH
	- PUCCH leva controle
	- PUSCH leva dados
	- PRACH participa do acesso aleatório
### Por que essa estrutura é importante
- A estrutura de canais permite separar diferentes tipos de informação
- Isso melhora a organização da transmissão
- Também permite que a rede trate controle e dados de forma diferente
- O mapeamento eficiente ajuda a garantir qualidade de serviço, escalabilidade e eficiência espectral
- Essa hierarquia é essencial para atender serviços com requisitos muito variados
- Exemplos
	- Aplicações críticas precisam de baixa latência
	- Streaming precisa de alta taxa de dados
	- IoT massivo precisa de suporte a muitos dispositivos
### PUCCH e informações de controle no uplink
- O PUCCH significa Physical Uplink Control Channel
- Ele é um canal físico usado no **uplink**, ou seja, no sentido UE → rede
- Sua função principal é transportar informações de controle, não dados grandes do usuário
- Ele ajuda a rede a saber se a comunicação está funcionando bem e se o UE precisa de recursos para transmitir
### CQI
- CQI significa Channel Quality Indicator
- É uma informação enviada pelo UE para indicar a qualidade do canal de rádio
- A rede usa o CQI para escolher a melhor modulação, codificação e taxa de transmissão
- Se o canal estiver bom, a rede pode usar uma modulação mais alta
- Se o canal estiver ruim, a rede usa uma modulação mais robusta

### ACK e NACK
- ACK significa confirmação positiva
- Indica que o dado foi recebido corretamente
- NACK significa confirmação negativa
- Indica que o dado chegou com erro ou não foi decodificado corretamente
- Quando ocorre NACK, a rede pode retransmitir o dado
- Isso ajuda a aumentar a confiabilidade da comunicação
### SR
- SR significa Scheduling Request
- É uma solicitação de agendamento enviada pelo UE para a rede
- O UE usa o SR quando precisa transmitir dados no uplink, mas ainda não tem recursos alocados
- A rede recebe essa solicitação e pode liberar recursos para o UE transmitir pelo PUSCH
### Diferença rápida entre PUCCH e PUSCH
- **PUCCH**
    - Canal físico de controle no uplink
    - Transporta CQI, ACK, NACK e SR
    - Serve para sinalização de controle
- **PUSCH**
    - Canal físico de dados no uplink
    - Transporta os dados reais do usuário
    - Pode carregar internet, arquivos, voz, vídeo e dados de aplicações
### Conceito que eu preciso lembrar
- PUCCH leva controle no uplink
- CQI informa a qualidade do canal
- ACK/NACK confirma se o dado foi recebido corretamente ou não
- SR pede recurso para transmitir no uplink
- PUSCH é usado quando o UE realmente vai enviar os dados
### Conclusão
- A estrutura de canais do 5G NR é organizada em canais lógicos, canais de transporte e canais físicos
- Os canais lógicos indicam o tipo de informação transportada
- Os canais de transporte definem como os dados serão levados pela interface de rádio
- Os canais físicos fazem a transmissão real no meio sem fio
- No downlink, aparecem canais como PBCH, PDSCH e PDCCH
- No uplink, aparecem canais como PUSCH, PUCCH e PRACH
- Essa organização torna o 5G mais flexível, eficiente e capaz de atender diferentes aplicações
- Resumindo
	- Canais lógicos dizem o que está sendo transportado
	- Canais de transporte dizem como será transportado
	- Canais físicos transmitem de fato pelo rádio
	- Downlink é rede para usuário
	- Uplink é usuário para rede
	- PDCCH e PUCCH são canais de controle
	- PDSCH e PUSCH são canais de dados
	- PRACH é usado no acesso aleatório
	- O mapeamento correto dos canais ajuda o 5G a entregar desempenho, confiabilidade e eficiência
