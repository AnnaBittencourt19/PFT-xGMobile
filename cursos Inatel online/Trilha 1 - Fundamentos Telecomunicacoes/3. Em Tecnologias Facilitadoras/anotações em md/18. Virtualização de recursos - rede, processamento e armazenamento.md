### Fundamentos em Telecomunicações em Tecnologias Facilitadoras
- Fundamentos de infraestrutura de TI
- Tema principal dessa apresentação: virtualização de recursos de computação, rede e armazenamento, mostrando como a virtualização entra entre o hardware e o sistema operacional para permitir maior flexibilidade, isolamento, eficiência e uso compartilhado dos recursos
### Virtualização de recursos
- Virtualização é o processo de transformar recursos físicos em recursos lógicos
- Em vez de uma aplicação depender diretamente de um hardware físico específico, ela passa a usar uma camada virtualizada
- Essa camada fica entre o hardware e o sistema operacional
- A apresentação mostra que, no contexto de telecomunicações e principalmente com o 5G, os recursos fornecidos pelos componentes do data center são virtualizados antes de serem compartilhados com as funções de rede
- Isso significa que processamento, armazenamento e rede deixam de ser consumidos de forma rígida e passam a ser entregues de forma mais flexível
### Ideia principal da virtualização
- Em uma infraestrutura tradicional, o sistema operacional costuma rodar diretamente sobre o hardware
- Esse modelo cria uma relação mais direta entre máquina física e sistema operacional
- Na virtualização, é adicionada uma camada intermediária
- Essa camada permite criar várias máquinas virtuais sobre o mesmo hardware físico
- Cada máquina virtual pode executar seu próprio sistema operacional e suas próprias aplicações
- Assim, um único servidor físico pode hospedar vários ambientes independentes
### Camada de virtualização
- A camada de virtualização fica entre o hardware e o sistema operacional
- Ela é responsável por abstrair os recursos físicos e apresentar recursos lógicos para as máquinas virtuais
- Em vez de a aplicação enxergar diretamente CPU, memória, disco e rede físicos, ela enxerga recursos virtualizados
- Essa camada permite melhor aproveitamento do data center
- Também permite que funções de rede sejam implantadas com mais agilidade
- Em telecomunicações, isso é muito importante porque as redes modernas precisam escalar rapidamente e suportar diferentes serviços
### Relação com a infraestrutura de TI
- No material anterior, a infraestrutura de TI foi apresentada com camadas como data center, processamento, armazenamento, rede e sistema operacional
- Nesta apresentação, aparece um novo elemento entre os recursos físicos e o sistema operacional: a virtualização
- A estrutura fica assim:
	- Data center
	- Processamento, armazenamento e rede
	- Virtualização
	- Sistema operacional
	- Aplicações, serviços e funções de rede
- Essa mudança é importante porque permite que os recursos do data center sejam compartilhados de forma mais eficiente
### Por que a virtualização é importante em telecomunicações?
- As redes de telecomunicações modernas precisam atender muitos usuários, muitos dispositivos e muitos tipos de serviço
- Com 5G, a rede precisa suportar aplicações com requisitos diferentes
- Exemplos:
	- Baixa latência
	- Alta disponibilidade
	- Grande volume de dados
	- Conectividade massiva
	- Funções de rede distribuídas
	- Serviços críticos
- A virtualização permite que funções de rede sejam criadas, movidas, escaladas e gerenciadas com mais flexibilidade
- Isso reduz dependência de equipamentos físicos dedicados
### Recursos virtualizados
- Os principais recursos que podem ser virtualizados são:
	- Processamento
	- Armazenamento
	- Rede
- Virtualizar processamento significa permitir que CPU e memória de um servidor físico sejam compartilhadas entre várias máquinas virtuais
- Virtualizar armazenamento significa criar volumes, discos e espaços lógicos independentes do dispositivo físico
- Virtualizar rede significa criar redes, interfaces, switches e conexões lógicas por software
- Esses recursos combinados formam a base de data centers modernos, cloud computing e ambientes de telecomunicações virtualizados
### Virtualização de computação
- Virtualização de computação também pode ser chamada de virtualização de servidor ou computação definida por software
- Ela permite que múltiplos sistemas operacionais sejam executados no mesmo hardware
- Cada sistema operacional roda dentro de uma máquina virtual
- As máquinas virtuais compartilham os recursos físicos, mas funcionam como ambientes separados
- Exemplo:
	- Um servidor físico possui CPU, memória e disco
	- A virtualização divide esses recursos
	- Uma VM roda Linux
	- Outra VM roda Windows
	- Outra VM roda um serviço específico de rede
- Tudo isso ocorre no mesmo hardware físico
### Máquina virtual
- VM significa Virtual Machine
- Em português, máquina virtual
- Uma máquina virtual é um ambiente lógico que simula um computador
- Ela possui recursos como CPU, memória, disco e interface de rede virtuais
- Dentro dela pode ser instalado um sistema operacional
- Para a aplicação, a VM se comporta como se fosse uma máquina real
- Porém, na prática, ela está usando recursos físicos compartilhados por meio da camada de virtualização
### Sistema operacional na virtualização
- SO significa Sistema Operacional
- Cada máquina virtual pode ter seu próprio sistema operacional
- Isso permite executar diferentes ambientes no mesmo servidor físico
- Exemplo:
	- VM 1 com Linux para aplicação web
	- VM 2 com Windows Server para aplicação corporativa
	- VM 3 com Linux para banco de dados
- A vantagem é que cada VM funciona de forma separada, mesmo usando o mesmo hardware físico
### Arquitetura básica de virtualização
| Camada | Função |
|---|---|
| Hardware | Fornece recursos físicos como CPU, memória, disco e rede |
| Virtualização | Abstrai os recursos físicos e cria ambientes virtuais |
| VM | Máquina virtual criada sobre a camada de virtualização |
| SO | Sistema operacional instalado dentro da VM |
| Aplicação | Serviço executado dentro do sistema operacional |
### Compatibilidade e flexibilidade
- A virtualização aumenta a compatibilidade porque permite executar diferentes sistemas operacionais no mesmo hardware
- Também aumenta a flexibilidade porque os recursos podem ser alocados conforme a necessidade
- Em vez de comprar um servidor físico para cada aplicação, é possível criar várias VMs no mesmo servidor
- Isso melhora o uso dos recursos e facilita mudanças na infraestrutura
- Se uma aplicação precisa de mais memória ou CPU, a VM pode ser ajustada
- Se um serviço precisa ser replicado, novas VMs podem ser criadas
### Isolamento da aplicação
- A apresentação destaca que a virtualização isola a aplicação dos recursos do sistema operacional e também de outras aplicações
- Isso significa que uma aplicação não precisa depender diretamente do hardware físico
- Também significa que uma aplicação em uma VM não interfere diretamente em outra VM
- Esse isolamento melhora segurança e estabilidade
- Se uma VM apresentar falha, as outras podem continuar funcionando
### Desacoplamento de recursos
- Desacoplamento significa separar a aplicação e o sistema operacional do hardware físico específico
- Antes, uma aplicação podia depender de um servidor específico
- Com virtualização, a aplicação roda em uma VM, e essa VM pode ser movida entre servidores físicos
- Isso facilita manutenção, migração e balanceamento de recursos
- A camada que faz essa abstração é compreendida pelo hypervisor
- Cada máquina virtual opera com seu próprio sistema operacional e seus aplicativos, independentemente do hardware físico direto
### Migração de máquinas virtuais
- Uma consequência importante do desacoplamento é a possibilidade de migrar VMs entre servidores físicos
- Em muitos cenários, essa migração pode ocorrer sem interrupção do serviço
- Isso é muito útil para manutenção e continuidade operacional
- Exemplo:
	- Um servidor físico precisa passar por manutenção
	- As VMs podem ser movidas para outro servidor
	- O serviço continua funcionando
- Esse recurso melhora disponibilidade e reduz impacto de falhas ou manutenções programadas
### Isolação de recursos
- Isolação significa que as máquinas virtuais são separadas umas das outras
- Uma falha em uma VM não deve afetar diretamente as demais
- Um ataque em uma VM também não deve comprometer automaticamente todas as outras
- Isso melhora a segurança e a estabilidade do ambiente
- Na prática, várias VMs podem funcionar no mesmo servidor físico, mas com ambientes independentes
### Ambientes de desenvolvimento, teste e produção
- A isolação permite criar ambientes diferentes no mesmo hardware
- Exemplos:
	- Desenvolvimento
	- Teste
	- Homologação
	- Produção
- Cada ambiente pode ter suas próprias configurações, aplicações e permissões
- Isso facilita o gerenciamento e reduz custo
- A equipe pode testar uma mudança em um ambiente separado antes de aplicar em produção
- Assim, a virtualização ajuda a evitar que testes afetem serviços reais
### Benefícios da virtualização
- A virtualização traz vários benefícios para infraestrutura de TI
- Principais benefícios:
	- Melhor eficiência no uso do hardware
	- Maior flexibilidade
	- Redução de custos
	- Manutenção simplificada
	- Maior compatibilidade
	- Isolamento entre ambientes
	- Melhor segurança
	- Migração de VMs
	- Melhor uso do data center
	- Facilidade para criar ambientes novos
### Eficiência
- A virtualização melhora a eficiência porque evita subutilização de servidores físicos
- Em muitos ambientes tradicionais, um servidor físico roda apenas uma aplicação e fica com recursos sobrando
- Com virtualização, vários serviços podem compartilhar o mesmo hardware
- Isso aumenta o aproveitamento de CPU, memória, armazenamento e rede
- Como resultado, menos servidores físicos podem atender mais serviços
### Redução de custos
- A redução de custos acontece porque a virtualização permite consolidar servidores
- Consolidar significa reunir várias cargas de trabalho em menos equipamentos físicos
- Isso pode reduzir:
	- Compra de hardware
	- Consumo de energia
	- Espaço físico
	- Refrigeração
	- Manutenção
	- Tempo de implantação
- Em data centers, esses custos são muito relevantes
### Manutenção simplificada
- A manutenção fica mais simples porque as VMs podem ser movidas, copiadas, pausadas ou recriadas com mais facilidade
- Também é possível criar snapshots
- Snapshot é uma captura do estado de uma VM em determinado momento
- Isso ajuda em testes, atualizações e recuperação de falhas
- Se algo der errado após uma atualização, a VM pode voltar para um estado anterior
### Hypervisor
- Hypervisor é o componente essencial no processo de virtualização
- Ele permite criar e gerenciar máquinas virtuais
- É o software ou camada que controla como os recursos físicos serão divididos entre as VMs
- O hypervisor gerencia:
	- CPU
	- Memória
	- Disco
	- Rede
	- Isolamento
	- Execução das VMs
- Sem hypervisor, não existe virtualização de máquinas virtuais nesse modelo
### Tipos de hypervisor
- A apresentação divide o hypervisor em dois tipos:
	- Bare-metal
	- Hosted
### Hypervisor bare-metal
- Bare-metal é o hypervisor que roda diretamente sobre o hardware
- Nesse tipo, não há necessidade de um sistema operacional tradicional abaixo das máquinas virtuais
- O hypervisor é instalado diretamente no servidor físico
- Vantagens:
	- Melhor desempenho
	- Maior controle sobre o hardware
	- Mais usado em data centers e ambientes corporativos
	- Mais adequado para produção
- Exemplos:
	- VMware ESXi
	- Microsoft Hyper-V em modo servidor
	- Xen
	- KVM
### Hypervisor hosted
- Hosted é o hypervisor que roda sobre um sistema operacional já instalado
- O sistema operacional da máquina física é chamado de host
- O sistema operacional dentro da máquina virtual é chamado de guest
- Nesse modelo:
	- Hardware
	- Sistema operacional host
	- Hypervisor/aplicação de virtualização
	- Máquinas virtuais com sistemas guests
- Vantagens:
	- Mais simples para uso em computadores pessoais
	- Bom para testes, estudos e laboratórios
	- Fácil de instalar
- Exemplos:
	- VMware Workstation
	- VirtualBox
	- Parallels Desktop
### Host e guest
- Host é o sistema operacional principal da máquina física
- Guest é o sistema operacional que roda dentro da máquina virtual
- Exemplo:
	- Um notebook com macOS como host
	- Uma VM com Linux como guest
	- Outra VM com Windows como guest
- Esse modelo é comum em ambientes de teste e desenvolvimento
### Bare-metal x hosted
| Tipo | Onde roda | Vantagem | Uso comum |
|---|---|---|---|
| Bare-metal | Diretamente no hardware | Melhor desempenho e controle | Data centers e produção |
| Hosted | Sobre um sistema operacional host | Simplicidade e facilidade de uso | Estudos, testes e desktops |
### Plataformas de virtualização
- Uma plataforma de virtualização é um conjunto de tecnologias e ferramentas que usam o hypervisor para criar um ambiente completo de virtualização
- Ela não faz apenas a criação de VMs
- Também pode incluir recursos de gerenciamento, rede, armazenamento, segurança, orquestração e automação
- Em ambientes corporativos, a plataforma de virtualização é importante porque permite gerenciar muitas máquinas virtuais de forma organizada
### Recursos de uma plataforma de virtualização
- Uma plataforma de virtualização pode incluir:
	- Gerenciamento de máquinas virtuais
	- Armazenamento virtualizado
	- Rede virtualizada
	- Segurança
	- Orquestração
	- Automação
	- Monitoramento
	- Migração de VMs
	- Alta disponibilidade
- Isso transforma a virtualização em uma solução completa de infraestrutura
### Exemplos de hypervisors e plataformas
- Exemplos citados na apresentação:
	- VMware ESXi
	- VMware Workstation
	- Hyper-V
	- Xen
	- VirtualBox
	- Parallels Desktop
	- KVM
	- VMware vSphere
	- Microsoft System Center
	- OpenStack
	- Red Hat Virtualization
- Alguns nomes aparecem como hypervisors
- Outros aparecem como plataformas de gerenciamento e virtualização
### VMware ESXi
- VMware ESXi é um hypervisor bare-metal
- É muito usado em ambientes corporativos
- Ele roda diretamente no hardware do servidor
- Permite criar e gerenciar máquinas virtuais com bom desempenho
### VMware vSphere
- VMware vSphere é uma plataforma de virtualização
- Ela usa o ESXi como base e adiciona recursos de gerenciamento
- Permite administrar ambientes maiores, com múltiplos servidores e VMs
- É usada em data centers corporativos
### Hyper-V
- Hyper-V é a tecnologia de virtualização da Microsoft
- Pode ser usada em ambientes Windows Server e também em algumas versões de Windows para desktop
- Permite criar e gerenciar VMs
- Pode ser usado tanto em ambientes corporativos quanto em laboratórios
### KVM
- KVM significa Kernel-based Virtual Machine
- É uma tecnologia de virtualização baseada no kernel do Linux
- É muito usada em ambientes Linux, cloud e plataformas abertas
- Também pode ser usada como base para soluções como OpenStack
### OpenStack
- OpenStack é uma plataforma aberta para criação e gerenciamento de infraestrutura em nuvem
- Ela pode usar hypervisors como KVM, Xen, Hyper-V ou VMware ESXi
- É usada para criar ambientes de cloud privada ou infraestrutura como serviço
- Em telecomunicações, OpenStack aparece bastante em cenários de NFV e cloud privada
### Virtualização e 5G
- No 5G, muitas funções de rede podem ser implementadas como software
- Isso exige infraestrutura capaz de criar, escalar e mover funções de forma dinâmica
- A virtualização permite que funções de rede sejam executadas em servidores comuns, em vez de dependerem sempre de equipamentos dedicados
- Isso torna a rede mais flexível e programável
- A virtualização também prepara o caminho para NFV, cloud, edge computing e redes definidas por software
### Relação com NFV
- NFV significa Network Function Virtualization
- Em português, virtualização de funções de rede
- A ideia é transformar funções de rede em software executado sobre infraestrutura virtualizada
- Exemplos de funções que podem ser virtualizadas:
	- Firewall
	- Roteador
	- Gateway
	- Balanceador
	- Funções do núcleo 5G
- A virtualização de recursos é uma base para que NFV funcione
### Relação com ICT
- ICT significa Information and Communication Technology
- Em português, tecnologias da informação e comunicação
- A conclusão da apresentação cita que a transição para sistemas ICT amplia as capacidades e a interconexão dos sistemas de TI
- Isso mostra que TI e telecomunicações estão cada vez mais integradas
- Virtualização é um dos elementos que permite essa integração
- Ela aproxima data centers, cloud, telecomunicações e funções de rede em um mesmo modelo operacional
