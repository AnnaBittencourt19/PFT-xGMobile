- Enquanto a codificação de fonte comprime os dados, a codificação de canal protege os dados contra erros causados pelo canal (ruído, interferência, desvanecimento, etc)
- Codificação de canal é a técnica que adiciona redundância controlada aos bits antes de transmitir
- Permite que o receptor detecte e em muitos casos corrija erros introduzidos pelo canal, sem precisar pedir retransmissão
- Receptor pode reconstruir
- Códigos para detecção de erros que serão tratados nesse material
### Importância da codificação de canal
- Eficiência do uso de banda
	- Permite usar modulações mais densas (ex: 64-QAM, 256-QAM) mesmo em canais ruins porque os erros podem ser corrigidos
- Desempenho em ambientes adversos
	- Essencial em satélite, celular em movimento, áreas com muita interferência
- Correção de erros
	- O receptor consegue corrigir erros sem pedir retransmissão
### Dois tipos de códigos
- Códigos para detecção de erros
	- Função: Só identifica que houve erro
	- Quando usar: Quando é fácil pedir retransmissão
- Códigos para detecção + correção
	- Função: Detecta e corrige os erros automaticamente
	- Quando usar: Quando retransmissão é difícil (satélite, móvel)
### Formas usuais de controle de erro
- ARQ – Automatic Repeat Request (detecção + retransmissão)
	- Transmissor envia pacote + bits de verificação (CRC ou paridade)
	- Receptor verifica se estiver OK envia ACK (confirmação)
	- ACK: Quando o receptor verifica o pacote e não encontra erro ele envia um ACK para o transmissor (ok pode mandar o próximo)
	- Se erro -> envia NACK -> transmissor retransmite o pacote (se tiver erro)
	- NACK: Receptor recebe com erro e pede para que haja a retransmissão
	- Muito usado em redes de computadores (TCP, Wi-Fi)
- FEC – Forward Error Correction (correção direta)
	- Transmissor adiciona muita redundância (códigos Hamming, Reed-Solomon, LDPC, convolucionais)
	- Receptor corrige os erros sozinho, sem pedir retransmissão
	- Usado em 5G, satélite, TV digital, armazenamento (SSD)
### Códigos para detecção de erros: Paridade simples
Método mais simples:
- Adiciona 1 bit extra no final do byte
- Paridade par: número total de “1” deve ser par 
- Paridade ímpar: número total de “1” deve ser ímpar 
Exemplo no slide:
- 10110010 → paridade par = 0 (total de 1s fica par)(adiciona 0 no final) (4 1s)
- 10110010 → paridade ímpar = 1  (adiciona 1 no final)(5 1s)
### Paridade Bidimensional (2D)
- Mais robusta
- Organiza os bits em uma matriz (linhas × colunas)
- Calcula paridade horizontal (por linha) -> LRC
- Calcula paridade vertical (por coluna) -> VRC
- Vantagem:
	- Detecta e corrige erro de 1 bit
	- Detecta erros de até 3 bits em alguns casos
### CRC (Cyclic Redundancy Check)
- O método mais usado hoje (Ethernet, Wi-Fi, 4G/5G, USB, etc.).
- Trata os bits como um polinômio
- Usa um polinômio gerador fixo (ex: 1011)
- Faz divisão polinomial em módulo 2 (só XOR)
- O resto da divisão é o FCS (Frame Check Sequence) que é enviado junto
- Vantagem: detecta múltiplos erros com altíssima probabilidade
### Como funciona o CRC (matemática)
1. Mensagem M(x) vira polinômio
2. Multiplica por x^r (adiciona r zeros à direita)
3. Divide por G(x) → resto = FCS
4. Transmite M(x)·x^r – FCS (exatamente divisível por G(x))

- No receptor: divide tudo por G(x).
- Resto = 0 -> sem erro
- Resto ≠ 0 -> erro detectado

- Codificação de canal = adicionar redundância para combater erros
- ARQ = detecção + retransmissão
- FEC = correção direta
- Técnicas simples: paridade, paridade 2D
- Técnica avançada: CRC (padrão atual)
- Codificação de canal = o “escudo” contra o ruído do canal
- Detecção = avisa que deu erro
- Correção = conserta o erro sozinho
- CRC = o método mais usado no mundo real
