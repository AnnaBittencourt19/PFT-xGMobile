- Classificação binária: Classificar quem é a classe 1 e quem é a classe 2 e separa-los, classe 1 com classe 1 e classe 2 com classe 2
- Classificação multiclasse: Varias classes
- Multirrótulo: Quando tem mais de um rótulo (mais de uma saída do treinamento)
- Ordinal: Quando as classes tem uma ordem ordinal (é ordenado)
### Classificação binária
- Verifica qual classe a entrada pertence
- Dados rotulados
### Regressão Logística
- Algoritmo de classificação, classe 0 ou classe 1
- Por baixo dos panos: primeiro calcula uma reta (linear), parte logística calcula a probabilidade da entrada fazer parte de classe (aí que entra a classificação)
- Como funciona:
	```
	1º) Valor linear (z): 
	z = w1*x1 + w2*x2 + b
	
	2º) Passa valor linear para função sigmoide
	```
- N é a quantidade de exemplos
- Quando soma np.array([-2, -2]) os pontos ficam centralizados em (-2, -2)
- $w$ é peso do treinamento (w1 e w2 os pesos de cada entrada)
	- As entradas também são chamadas de features
	- Os pesos indicam o quanto cada variável influencia na decisão final
- $b$ é o bias (vies), ajuste da reta
	- O bias ajuda o modelo a ajustar a separação entre as classes
- Sigmoide: Transforma qualquer valor real em um número entre 0 e 1 (faz a função probabilidade resultar entre 0 e 1)
	- Se for > 0.5: É da classe 1
	- Se for < 0.5: Não é da classe 1 e sim da classe 0
- Classe 0 e Classe 1: São "pontos" definidos por nós
- Matrizes, transforma em vetores para jogar em uma matriz só 
- Ravel(): Achatamento, transforma em vetor
- np.vstack junta os dados verticalmente(juntar xs da classe 0 e 1)
- np.hstack junta os rótulos horizontalmente(juntar ys da classe 0 e 1)
- Tem dados de treino (x_train, y_train) e dados de teste (x_test e y_test). Uma parte dos dados fica pra treino e outra para teste
	- train_test_split: usado para separar os dados em treino e teste
- model.fit treina o modelo
	- O modelo aprende os melhores valores de w e b durante o treinamento
- model.predict faz a previsão da classe
- model.predict_proba mostra a probabilidade de cada classe
- random_state  serve para manter o mesmo resultado quando o código for rodado de novo (reprodutibilidade)
