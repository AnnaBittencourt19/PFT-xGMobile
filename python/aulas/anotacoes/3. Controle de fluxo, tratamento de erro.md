### Comandos de interrupção
- break: sai do for, assim que chega no break sai do break, quando tem um for dentro de outro ele sai de todos os for's
- continue: pula, ignora a iteração atual e vai para a próxima
- Pass: nula (para caso queira implementar algo no futuro)
### Tratamento de exceções
- Try: Onde fica o código que pode dar erro, tipo uma divisão que pode ser por 0 e resultar um erro (deve ser sempre acompanhado de except ou finally, deve acontecer alguma coisa com o erro)
- Except: Captura o erro
- Else: Executa se o código não retornar erro (só em caso de sucesso)
- Finally: Fim
- Exception python: mostra erros mais comuns (exemplo: ZeroDivisionError)
### Criar o próprio erro
- Raise: Customiza um novo erro
- Classes de erros (novo erro):
```Python
class SaldoInsuficienteError(Exception): #Exception: Erro técnico
	pass
```
- Chamar a classe do erro:
```Python
raise SaldoInsuficienteError("Saldo baixo")
```
	- Raise dá o erro automaticamente e para imediatamente
